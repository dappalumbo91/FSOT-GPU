**3D spatial architecture is not dismissed.** Spatial reasoning will need explicit spatial axes. Order of work:

1. **Now:** exhaust lawful DoF inside the **current host geometry** (sequence × head × channel × layer-depth, FSOT consensus, full trainable body under pure FSOT attention), with **D_eff-aware** scalar/LR/gates where they already exist in the stack.

If current geometry already supplies the connective DoF we need for LM host SOTA, we save work and port to 3D when spatial tasks land.
